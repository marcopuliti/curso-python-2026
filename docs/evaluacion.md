# Sistema de evaluación

## Condiciones de aprobación

Para aprobar el curso se requiere:

1. **Asistencia mínima del 75 %** de las clases presenciales.
2. **Entrega y aprobación de las guías de ejercicios** de cada módulo (evaluación continua).
3. **Aprobación del proyecto final** con su presentación oral (ver [proyecto-final.md](proyecto-final.md)).

## Evaluación continua: guías de ejercicios

- Cada módulo tiene una guía de ejercicios prácticos que se resuelve **individualmente**.
- Las entregas son **semanales**: la guía del módulo se entrega antes del inicio del módulo siguiente (fechas exactas se publican en clase).
- Formato de entrega: notebook `.ipynb` con los ejercicios resueltos. A partir del módulo 5, las entregas se hacen subiendo el notebook al repositorio personal de GitHub.
- Criterio de aprobación de cada guía: al menos el **60 % de los ejercicios resueltos correctamente**. Las guías no aprobadas pueden reentregarse una vez, dentro de la semana siguiente a la devolución.

## Uso de asistentes de IA en las guías

- **Módulos 1 a 4:** el objetivo es que desarrolles las habilidades de base. Podés usar IA para *entender* conceptos o errores, pero el código entregado debe ser escrito por vos. Declaralo si la usaste.
- **Módulos 5 y 6 y proyecto final:** el uso de IA asistida es **parte del contenido del curso** y está no solo permitido sino requerido en el proyecto, siempre con revisión crítica del código generado y declaración de qué herramientas se usaron y cómo.

## Registro de asistencia

La asistencia se registra en cada clase. Con 9–10 semanas de clases, el 75 % admite como máximo 2 a 3 inasistencias.
