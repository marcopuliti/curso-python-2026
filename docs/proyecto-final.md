# Proyecto final

## Consigna

Desarrollar un **script o pequeña aplicación en Python relacionada con tu área de investigación**, que cumpla con:

1. **Relevancia:** resuelve un problema real (aunque sea pequeño) de tu trabajo de tesis o investigación: automatizar un cálculo, procesar archivos de datos, generar figuras, simular un modelo simple, etc.
2. **Uso de IA asistida:** durante el desarrollo debe usarse **al menos una herramienta de IA** (GitHub Copilot, ChatGPT, Claude u otra), con revisión crítica del código generado.
3. **Publicación en GitHub:** el proyecto se entrega como repositorio público (o privado con acceso a los docentes) en GitHub.
4. **Presentación oral:** se presenta en la última clase (**15/11/2026**), 10 minutos por estudiante + 5 de preguntas.

## Entregables

El repositorio de GitHub debe contener:

- **El código** (`.py` y/o `.ipynb`), organizado y con nombres claros.
- **`README.md`** con: qué problema resuelve, cómo se instala/ejecuta, ejemplo de uso, y una sección **"Uso de IA"** que declare qué herramientas usaste, para qué, y qué tuviste que corregir del código generado.
- **Datos de ejemplo** (o instrucciones para obtenerlos) que permitan ejecutar el proyecto.
- **Historial de commits** que refleje el proceso de desarrollo (no un único commit final).

## Alcance esperado

No se espera software profesional: se espera un proyecto **pequeño pero completo y funcional**. Referencia: entre 100 y 400 líneas de código propio, desarrollado en 2 a 3 semanas a ritmo de posgrado. Ideas típicas por área:

- **Física/Química:** procesar salidas de un instrumento o simulación y graficar resultados.
- **Matemática:** implementar y visualizar un algoritmo o modelo numérico.
- **Biología/Geología:** limpiar y resumir planillas de datos de campo, generar figuras.
- **Computación/Electrónica:** herramienta de línea de comandos para una tarea repetitiva.

El tema se acuerda con los docentes **antes del 31/10/2026** (basta un párrafo por correo o en clase).

## Rúbrica de evaluación

| Criterio | Peso | Excelente (100 %) | Suficiente (60 %) | Insuficiente (0 %) |
|---|---|---|---|---|
| **Funcionamiento** | 30 % | El código corre sin errores y resuelve el problema planteado | Corre con ayuda menor o resuelve parcialmente | No funciona |
| **Calidad del código** | 20 % | Funciones bien organizadas, nombres claros, comentarios donde hace falta | Código funcional pero desordenado | Ilegible o copiado sin comprensión |
| **Uso del repositorio Git** | 15 % | Commits frecuentes y descriptivos, README completo | Pocos commits, README mínimo | Un solo commit o sin README |
| **Uso crítico de IA** | 15 % | Declara el uso, identifica errores del código generado y los corrige | Declara el uso sin análisis crítico | No declara o uso acrítico |
| **Presentación oral** | 20 % | Explica problema, solución y decisiones con claridad en el tiempo asignado | Explica el qué pero no el porqué | No puede explicar su propio código |

**Aprobación:** puntaje total ≥ 60 %, sin obtener "Insuficiente" en *Funcionamiento* ni en *Presentación oral*.
